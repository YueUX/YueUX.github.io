AI Manager 2.0
AI-driven enterprise chatbot CMS
CONTEXT

As part of Verizon’s “Enhancement of Customer Engagement” initiative, the omni-channel chatbot services needed an upgrade to extend the capabilities of what Verizon customers can do via the chat. The Chatbot service platform that the company used to build and maintain the conversational contents also faced an urgent need for a full upgrade. Hence, AI Manager 2.0 has been built to increase the technical cabilities, efficiency and usability.


ROLE & DURATION

UX / UI
End-to-end Design

Time     May 2018 - Present

PREVIOUS DESIGN

The previous platform (AI manager 1.0) supports basic content building, but it is very code-centric and not efficient.

  1. It has a very steep learning curve to be used by non-IT users (Screen 1).
  2. The way that each conversation intent is linked together was very tedious and not visual, which makes it very hard for the users to follow through even one simple flow (Screen 2).

COMPLEXITY #2 - Intertwined Conversation Components

In order to delivery the seamless user experience to the customers, the bot needs to be well-trained with NLP(Natual Language Processing). It has perform the system actions to go down the right path. It has to have the right jump dependencies. All these components are closely tied to each other.

Everytime we designed a new feature or iterate on existing ones, the design has to be made compatible with all rest parts.

COMPLEXITY #3 - Multiple user groups

The tool is facing multipe user groups and different team structures. Typical roles of a team include:

  • Conversation Designer (Business)
  • Conversation Designer (IT)
  • Lead Developer (IT)
  • NLP Specialist
  • QA
  • Domain admin
  • ...

(example workflow optimization)

The goal is to:

  • • improve the workflow so that the conversation can be quickly built out, tested, and published

SOLUTION - WYSISYG

The final design is a lego like visual interface. Users can drag and drop building blocks onto the canvas to build out the flow and configure each components.
